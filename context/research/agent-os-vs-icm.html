How this transfers to any real project

1

Drop a CLAUDE.md

Lean entry point. Identity + pointer to structure. This is the only file that's truly required. Everything else is optional layers you add when the project needs them.

2

Add context/

What the agent needs to know. Split stable config from live sources when it matters. Start with one folder. Grow as needed.

3

Add lists/

What needs to happen. A single markdown checklist is fine. Upgrade to YAML with metadata when the project outgrows a checklist.

4

Add actions/

What can be changed and how. Simple actions = a README. Repeatable pipelines = numbered stage folders (ICM pattern). Only add this layer when the project has write targets.
